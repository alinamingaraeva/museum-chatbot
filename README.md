# museum-chatbot
# Экономика ИИ-решения для чат-бота музея

## Сценарий использования ( сведения взяты из данных на сайте Jivo, где сейчас отвечают операторы руками) 
- Пользователей в месяц: 2 000
- Запросов в день (человек): 80
- Запросов на человека: 3–4 (в среднем 3,5)
- Запросов в день (всего): 280
- Запросов в месяц: 8 400
- Токенов на запрос: 350
- Месячный объём токенов: 2,94 млн

## Стоимость облачных API (в месяц)
| Модель | Стоимость |
|--------|-----------|
| Флагман (GPT-4.5 / Claude Opus) | ~$230 |
| Средний (Gemini Pro / Claude Sonnet) | ~$20 |
| Бюджетный (DeepSeek V3.2 / GPT-4o-mini) | **$2** |

## Локальное развёртывание
- Железо: RTX 3060 + сборка за $1 300
- Месячные расходы: $53 (электричество + амортизация)

## Точка безубыточности
- Локально vs бюджетное облако: **25,5 месяцев** (2 года 1,5 месяца)

## Выбранная стратегия
**Облако (бюджетная модель DeepSeek V3.2)**

Обоснование:
- Месячные расходы всего $2 — минимальны для музея
- Отсутствие CAPEX — не нужно покупать оборудование
- Качество бюджетной модели достаточно для ответов на вопросы об экскурсиях и продаже билетов
- При нагрузке 2 000 пользователей локальное решение не окупится за разумный срок
